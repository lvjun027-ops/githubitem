---

本指南提供分阶段的 Vibe Coding 提示词。学生可以按照阶段顺序，将每个步骤中的提示词直接复制给 AI 助手，逐步完成"AI Native 跨境电商智能运营平台"的开发。每个步骤都对应一段可直接使用的中文提示词，覆盖从环境搭建、基础框架到核心业务功能与 AI 能力的完整开发流程。建议初学者严格按阶段推进，先跑通基础功能，再逐步叠加智能化模块，最终构建出一个具备多语言、多模态、智能运营能力的完整跨境电商平台。

## 开发环境准备

本项目使用 Anaconda 管理 Python 依赖；数据库采用 Docker 中已安装并启动的 MySQL 8.4（端口 3306，账号 root，密码 123456）；使用 git 进行版本控制；前端开发依赖 nodejs；统一使用 vscode 作为开发工具。请在工作目录下创建项目总目录 `ai-global-ecommerce` 作为根目录，本项目所有前后端代码均放置在此目录中，以保持结构统一、便于协作与维护。

### 步骤4：后端认证API

**复制以下提示词到AI助手：**

```
请帮我实现完整的后端认证系统，包含JWT令牌管理、用户注册登录、密码加密与个人资料维护。

项目：AI Native 跨境电商智能运营平台
后端技术栈：Python + FastAPI + SQLAlchemy + PyMySQL + Pydantic + python-jose(JWT) + passlib(密码加密) + Redis(令牌缓存)
API统一前缀：/api/v1

后端API清单：
1. 注册 POST /api/v1/auth/register —— 接收 email、password、name、preferred_language(默认zh)、preferred_currency(默认CNY)、region(默认CN)；密码使用passlib bcrypt加密；默认role="user"；返回JWT token(有效期24h)
2. 登录 POST /api/v1/auth/login —— 接收 email+password；验证密码；返回JWT token + 用户信息(含preferred_language/preferred_currency/region)；登录失败返回401
3. 获取当前用户 GET /api/v1/auth/profile —— 需Bearer Token认证；返回当前用户完整信息(不含密码哈希)
4. 更新个人资料 PUT /api/v1/auth/profile —— 需认证；可更新name、preferred_language、preferred_currency、region；返回更新后用户信息
5. 修改密码 PUT /api/v1/auth/password —— 需认证；接收old_password+new_password；验证旧密码后更新；返回成功消息

核心安全设计：
- JWT生成使用python-jose，密钥从环境变量SECRET_KEY读取，算法HS256，有效期24小时
- 密码哈希使用passlib[bcrypt]，cost factor=12
- Token存入Redis白名单(key: token:{user_id}，TTL=24h)，登出时删除
- 所有需要认证的接口通过Depends(get_current_user)依赖注入获取当前用户
- Session使用SQLAlchemy异步会话，通过get_db依赖注入

请创建以下文件（含完整实现，所有路由注册到app/api/__init__.py的APIRouter）：
- backend/app/core/security.py —— JWT生成/验证、密码哈希/校验、get_current_user依赖、get_db依赖
- backend/app/schemas/auth.py —— Pydantic模型：UserRegister、UserLogin、UserResponse、UserUpdate、PasswordChange、TokenResponse
- backend/app/models/user.py —— SQLAlchemy模型：User表(id, email, name, hashed_password, role, preferred_language, preferred_currency, region, is_active, created_at, updated_at)
- backend/app/api/auth.py —— 上述5个API端点实现
- backend/app/services/auth_service.py —— 业务逻辑层：注册/登录/获取用户/更新资料/修改密码的服务函数
```

// frontend/src/utils/request.js
import axios from 'axios'
import { ElMessage } from 'element-plus'
import router from '@/router'

// 创建 Axios 实例，统一 baseURL 前缀
const service = axios.create({
  baseURL: '/api/v1',
  timeout: 15000
})

// 请求拦截器：自动携带 JWT
service.interceptors.request.use(
  (config) => {
    const token = localStorage.getItem('token')
    if (token) {
      config.headers['Authorization'] = `Bearer ${token}`
    }
    // 透传当前语言，便于后端返回本地化文案
    const locale = localStorage.getItem('locale')
    if (locale) {
      config.headers['Accept-Language'] = locale
    }
    return config
  },
  (error) => Promise.reject(error)
)

// 响应拦截器：处理 401 跳登录、统一错误提示
service.interceptors.response.use(
  (response) => {
    const res = response.data
    // 约定后端响应格式 { code, message, data }
    if (res.code !== undefined && res.code !== 0 && res.code !== 200) {
      ElMessage.error(res.message || '请求失败')
      return Promise.reject(new Error(res.message || 'Error'))
    }
    return res.data !== undefined ? res.data : res
  },
  (error) => {
    const status = error.response?.status
    if (status === 401) {
      // Token 失效，清理并跳转登录
      localStorage.removeItem('token')
      ElMessage.error('登录已过期，请重新登录')
      router.push('/login')
    } else {
      ElMessage.error(error.response?.data?.message || error.message || '网络异常')
    }
    return Promise.reject(error)
  }
)

export default service

    currency: currency || conf.currency
  }).format(amount || 0)

```javascript
// frontend/src/utils/sse.js
// EventSource 封装，用于 AI 购物助手的流式输出
export function createSSE(url, { onMessage, onDone, onError } = {}) {
  // 通过 query 透传 JWT（EventSource 不支持自定义 header）
  const token = localStorage.getItem('token')
  const fullUrl = `/api/v1${url}${url.includes('?') ? '&' : '?'}token=${token || ''}`
  const es = new EventSource(fullUrl)

  // 监听增量消息事件
  es.addEventListener('message', (e) => {
    if (e.data === '[DONE]') {
      es.close()
      onDone && onDone()
      return
    }
    onMessage && onMessage(e.data)
  })

  // 监听自定义 done 事件，结束流并关闭连接
  es.addEventListener('done', () => {
    es.close()
    onDone && onDone()
  })

  es.onerror = (err) => {
    es.close()
    onError && onError(err)
  }

  // 返回手动关闭方法，便于组件卸载时清理
  return { close: () => es.close() }
}
```

## 注意事项

1. **分阶段提交 Git**：每完成一个阶段就提交一次 Git，建议配合清晰的 commit message 与功能分支，确保进度可回溯、问题可定位。
2. **多语言文案统一维护**：所有界面文案必须统一在 `src/locales` 中维护，严禁在组件内硬编码文本，新增语言时只需扩展对应语言包。
3. **数据源保持同步**：ChromaDB/FAISS 向量库与 Elasticsearch 索引需与 MySQL 业务数据保持同步，商品、法规等数据变更后应及时触发 Celery 异步任务重建索引。
4. **可信数据源与定期更新**：汇率、税费、各国法规需对接权威可信的数据源，并设置定期更新机制，避免使用过期数据导致结算或合规错误。
5. **接口契约一致性**：前后端 API 路径统一走 `/api/v1` 前缀，响应格式保持 `{ code, message, data }` 约定一致，便于前端统一拦截处理。
6. **合规数据及时推送**：法规与合规规则一旦更新，应通过消息推送或缓存刷新机制及时同步到前端与各业务模块，确保跨境交易始终合规。
